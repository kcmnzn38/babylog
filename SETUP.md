# 🐣 じぶんの「◯◯ログ」を作る手順（だれでもOK版）

このアプリは、赤ちゃんのお世話記録（ミルク・睡眠・おむつ・写真など）を
**家族みんなのスマホで自動同期**しながら、**完全無料**で使えるWebアプリです。
このリポジトリは「べびログ」（はなちゃん用）ですが、コピーして**自分の子の名前のログ**を作れます。

かかる時間は約10分。プログラミングの知識は不要です。

## 用意するもの（すべて無料）

- **GitHubアカウント**（アプリの設計図のコピー置き場）… https://github.com/signup
- **Vercelアカウント**（アプリを動かす場所。GitHubアカウントでそのまま登録できます）… https://vercel.com/signup
- （任意）Googleアカウント … スプレッドシートへの自動バックアップを使う場合

## 手順1: アプリを自分のVercelに立てる

やり方は2つあります。**元のリポジトリが公開（Public）ならA**、
**zipファイルで受け取ったならB** に進んでください。どちらも結果は同じです。

### 方法A: デプロイボタンを押す（公開リポジトリの場合）

READMEの「**Deploy with Vercel**」ボタンをクリックすると、画面の案内が始まります。

1. **Create Git Repository**: リポジトリ名を決めます（例: 子が太郎くんなら `tarolog`）→ Create
2. **Add Environment Variables**: `APP_TOKEN` と `CRON_SECRET` の入力欄が出ます。
   家族で共有する**合言葉**（好きな英数字。例: `taro-family-2026`）を決めて、
   **2つとも同じ値**を入力 → Continue
3. **ストア作成**: Upstash（データベース）とBlob（写真置き場）の作成画面が出ます。
   - Upstash: プランは **Free**、リージョンは **Tokyo (hnd1)**、**EvictionはOFF**のまま
   - Blob: アクセスは **Private** のまま
4. **Deploy** を押して1〜2分待つと、あなた専用のURL（`https://◯◯.vercel.app`）が発行されます

### 方法B: zipをもらって自分のリポジトリから立てる

zipで受け取った場合はこちら。自分のリポジトリは**非公開（Private）のままでOK**です。

1. **GitHubに新しいリポジトリを作る**: https://github.com/new →
   名前を決めて（例: `tarolog`）、**Private** を選んで Create repository
2. **zipの中身をアップロード**: zipを展開 → 作ったリポジトリの画面で
   「uploading an existing file」リンク（または Add file → Upload files）→
   展開したフォルダの**中身を全部**ドラッグ&ドロップ → Commit changes
   （ファイルが多いのでアップロードに1分ほどかかります）
3. **Vercelに取り込む**: https://vercel.com/new →「Import Git Repository」で
   いま作ったリポジトリを選ぶ →
   **Environment Variables** を開いて `APP_TOKEN` と `CRON_SECRET` を追加
   （家族で共有する**合言葉**を決めて**2つとも同じ値**に）→ **Deploy**
4. **データベースをつなぐ**: デプロイ完了後、プロジェクト画面 → **Storage** タブ →
   - **Upstash for Redis** → Create: プランは **Free**、リージョンは **Tokyo (hnd1)**、EvictionはOFFのまま → Connect
   - **Blob** → Create: アクセスは **Private** → Connect
5. **Deployments** タブ → 最新のデプロイの「…」→ **Redeploy**
   （つないだデータベースの設定を反映させるため）

どちらの方法でも、データも写真もぜんぶ**あなた専用の無料枠**に保存される状態になります。
他の家族のデータと混ざることはありません。

## 手順2: スマホで使い始める

1. 発行されたURLをiPhone/AndroidのブラウザでOpen
2. ようこそ画面で赤ちゃんの名前・生年月日・性別を登録
3. 設定タブ →「同期パスコード」に手順1で決めた**合言葉**を入力
4. 共有メニューから「**ホーム画面に追加**」→ 以降はアプリとして起動できます

**家族の招待**は、URLと合言葉を教えて同じことをしてもらうだけ。
記録もプロフィールも自動で同期されます。

**見るだけの人（祖父母など）を招待したい場合**は、Vercelの環境変数に
`APP_TOKEN_VIEW=閲覧用の合言葉`（編集用とは別の値）を追加してRedeployし、
その合言葉を教えてください。閲覧用の合言葉で入った端末は記録・編集ができません。
同様に `APP_TOKEN_PHOTO=写真用の合言葉` を追加すると、
「閲覧＋写真の追加だけ」できる合言葉も作れます（預かり中に写真だけ送ってもらう用）。

## 手順3（任意）: スプレッドシートに毎日自動バックアップ

コードの書き換えは一切不要です（合言葉は初回の書き出しで自動登録されます）。

1. Googleスプレッドシートを新規作成 → メニュー「拡張機能」→「Apps Script」
2. 左の⚙「プロジェクトの設定」→
   「**「appsscript.json」マニフェスト ファイルをエディタで表示する**」にチェック
3. エディタのファイル一覧に出てきた `appsscript.json` を開き、
   このリポジトリの `gas/appsscript.json` の中身に**丸ごと差し替え**て保存
   （シート・Drive・外部通信の権限をまとめて宣言するファイルです。
   これを飛ばすと後で「権限がありません」エラーになります）
4. `コード.gs` に、このリポジトリの `gas/Code.gs` の中身を**丸ごと貼り付け**て保存
5. エディタ上部の関数選択（「doGet」などと出ているプルダウン）で
   「**setup**」を選んで ▶実行 → 承認画面が出たら:
   アカウント選択 →「このアプリはGoogleで確認されていません」→「詳細」
   →「（プロジェクト名）に移動」→ **許可**
   → 実行ログに「OK: 承認が完了しました」と出ればOK
6. 「デプロイ」→「新しいデプロイ」→ 種類「ウェブアプリ」→
   実行: 自分 / アクセス: 全員 → デプロイ → 発行されたURLをコピー
7. Vercelの画面 → プロジェクト → Settings → Environment Variables →
   `SHEET_WEBAPP_URL` という名前でそのURLを追加 → Deployments → Redeploy
8. アプリの設定タブ「書き出し先URL」にも同じURLを入れて、
   「**シートへ書き出す**」を1回押す
   → この初回書き出しで、アプリの合言葉がシート側のパスコードとして自動登録されます

これで**毎日 日本時間3:00** に全記録が日本語のシートに自動で書き出されます。

※あとからコードを更新するときは「デプロイ」→「デプロイを管理」→ 鉛筆 →
「**新バージョン**」で更新（「新しいデプロイ」にするとURLが変わってしまうので注意）。

### さらに任意: 写真をGoogle Driveにも自動バックアップ

権限は手順3の`setup`でまとめて承認済みなので、やることは2つだけです。

1. Google Driveでバックアップ先にしたい**親フォルダ**を作る（例: 「たろうの写真」）
2. フォルダを開いたときのURL（`https://drive.google.com/drive/folders/…`）を、
   アプリの設定タブ「写真バックアップ先のDriveフォルダ」にそのまま貼り付け

毎日の書き出しのタイミングで、写真が「写真」「うんち」など
**種類ごとのサブフォルダ**に自動保存されます。アプリの「シートへ書き出す」を
押すとその場で動作確認でき、失敗した場合は理由が画面に表示されます
（うまくいかないときは、`Code.gs`の`DRIVE_FOLDER_ID`にフォルダIDを入れて
エディタから`testDriveFolder`を実行すると診断できます）。

## 手順4（任意）: Siriに話して記録（Gemini API）

「Hey Siri、べびログ」→「ミルク120飲んだ」「30分前にうんち多め」のように、
**アプリを開かずに話すだけで記録**できます。AIが自然な言葉を解釈するので言い方は自由。
ロック画面やApple Watchからも使えます。

**準備（1回だけ・無料）:**

1. https://aistudio.google.com/apikey でGoogleアカウントでログイン →「APIキーを作成」
2. Vercelの Settings → Environment Variables に `GEMINI_API_KEY` としてそのキーを追加
3. Deployments → Redeploy

**ショートカットを作る:**

iPhoneの「ショートカット」アプリで新規ショートカットを作り、アクションを4つ並べます:

1. **「テキストを音声入力」**
2. **「URLの内容を取得」** … URLに `https://あなたのアプリ.vercel.app/api/voice`、
   「方法」を **POST**、「ヘッダ」に `X-App-Token` ＝ 合言葉 を追加、
   「本文を要求」を **JSON** にしてフィールドを2つ:
   `text` ＝（変数「音声入力されたテキスト」）、`mode` ＝ `commit`
3. **「辞書の値を取得」** … キー `reply`（入力: URLの内容）
4. **「テキストを読み上げる」**（または「結果を表示」）

ショートカット名を「べびログ」にすれば「**Hey Siri、べびログ**」で起動できます。
※Siri経由は確認なしで即登録されます。まちがえたらアプリのタイムラインから削除してください。

## 手順5（任意）: アプリの名前を「◯◯ログ」に変える

GitHubの自分のリポジトリで、以下の2ファイルの「べびログ」を好きな名前に書き換えるだけです
（GitHubのWeb画面でファイルを開いて ✏️ 鉛筆マークから編集できます）。

- `public/index.html` … 「べびログ」をすべて置換（`<title>`・apple-mobile-web-app-title・
  ようこそ画面・画面下のバージョン表記）
- `public/manifest.webmanifest` … name / short_name

編集をコミットすると自動で再デプロイされ、ホーム画面の名前も変わります。

## 他アプリからの引っ越し

ぴよログの「メニュー → 記録の出力 → テキスト」で書き出したtxtを、
アプリの「設定 → ぴよログから取り込み」からファイル選択（複数可）するだけです。
同じデータを何度取り込んでも重複しません。

## よくある質問

- **お金はかかる？** … かかりません。Vercel・Upstash・Blobすべて無料枠内で、
  記録＋毎日の写真程度なら数年分余裕があります。
- **データはどこにある？** … あなたが作ったVercel/Upstashの中だけです。
  このリポジトリの作者を含め、他人はアクセスできません。
- **合言葉を変えたい** … Vercelの Environment Variables で `APP_TOKEN`・`CRON_SECRET` を変更
  →Redeploy→各端末の設定で新しい合言葉を入力（Apps Scriptを使っていればTOKENも変更）。
- **機種変更したら？** … 新しい端末でURLを開いて合言葉を入れるだけで全部戻ります。
- **写真だけ急に表示されなくなった** … Vercel無料枠（転送量）の一時制限の可能性があります。
  写真のデータは消えていません。アプリでは📷マークの表示になり、タップすると
  Driveバックアップへの案内が出ます。超過分が30日の窓から抜ければ自動で復活します。
  現在の設計は写真をCDNから直接配信して無料枠をほぼ使わないので、
  ふつうの家族利用でこの制限に当たることはまずありません。
- **祖父母に見せたいけど、間違って記録されるのが心配** … Vercelの環境変数に
  `APP_TOKEN_VIEW=閲覧用の合言葉` を追加してRedeployし、その合言葉を教えてください。
  閲覧用で入った端末は「👀 閲覧モード」になり、見ることはすべてできますが、
  記録・編集・削除はサーバー側で拒否されるのでできません。
